create(input, successCallback, errorCallBack)

KnurldSDK.appModel.create(inputs, successCallback, errorCallBack);
Parameters:
Name Type Description
input Object javascript object which contents all the inputs for API

e.g.

var inputs={};

inputs.enrollmentRepeats="enrollmentRepeats"; Required.

inputs.vocabulary="vocabulary"; Required.

inputs.verificationLength="verificationLength"; Required.

inputs.threshold=threshold; Optional.

inputs.autoThresholdEnable=autoThresholdEnable; Optional.

inputs.autoThresholdClearance=autoThresholdClearance; Optional.

inputs.autoThresholdMaxRise=autoThresholdMaxRise; Optional.

inputs.useModelUpdate=useModelUpdate; Optional.

inputs.modelUpdateDailyLimit=modelUpdateDailyLimit; Optional.

----------Info-----------

enrollmentRepeats:# of times the user is asked to repeat each phrase during the enrollment process. Minimum required is 3.

vocabulary:Consumers may enroll with any combination of the phrases associated with app-model. A subset of phrases will be selected for each verification.

verificationLength:# of phrases to use for verification. Recommended minimum is 3. Make sure that this is equal to or less than the number of phrases specified in the vocabulary parameter. The actual phrases asked for by the engine at verification time is a random subset of the phrase vocabulary.

threshold:Optional, Score threshold for verification. Default 2 (typical range between 2 and 4)

autoThresholdEnable:Optional, If enabled, consumers who constantly score high will have their threshold adjusted automatically. (optional, default true)

autoThresholdClearance:Optional, Distance above the base threshold that a consumer's score needs to be to activate auto-adjustment. (optional, default 2.5)

autoThresholdMaxRise:Optional, Maximum value above the base threshold that a consumer's auto-adjustment will rise to. (optional, default 1.0)

useModelUpdate:Optional, Optional. Enable augmenting enrollment data with audio from successful verifications. (optional, default false)

modelUpdateDailyLimit:Optional, Maximum amount of times per day that model updating will be applied (default 1)
successCallback function Success Call Back
errorCallBack function Error Call Back
